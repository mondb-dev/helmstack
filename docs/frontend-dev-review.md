# HelmStack — Front-End Development Review & Backlog

A curated, evaluated checklist for evolving HelmStack into a first-class **perception + execution layer for front-end development**.

Reviewed: 2026-06-18. Scope: `apps/desktop`, `packages/*`, `docs/*`.

**Legend** — Priority: `P0` ship-blocker / `P1` high / `P2` nice-to-have. Effort: `S` <1d / `M` ~days / `L` >1wk.

---

## 0. Strategic framing (read first)

HelmStack today is genuinely two products sharing one runtime:

1. **An autonomous web-agent substrate** — social-feed perception (`dom-extractor.ts` lines 276–917, ~640 lines), OAuth detection, encrypted account vault + TOTP, anti-detection/stealth (`anti-detection.ts`), human handoffs for captcha/2fa/payment.
2. **A front-end dev/test instrument** — console+network capture, visual diff, CWV, a11y audit, element-style inspector, component tree, responsive viewport suite, perception diff (`docs/dev-team-features.md`).

The **dev-tool half is the stated goal** and is already strong and well-architected. The agent-substrate half is largely orthogonal to front-end development and in places actively *fights* it (see §1 anti-detection jitter). This review treats the dev-tool as the product and the agent layer as the transport.

- [ ] **`P1`** Decide and document the positioning: is anti-detection / social perception / TOTP vault still in scope, or split into an optional plugin so the FE-dev core stays lean? Right now a developer pointing this at `localhost:3000` pays for (and is confused by) `social-feed` page-kind classification and stealth jitter they never asked for.

## 1. Critical fixes (correctness & security)

- [x] **`P0` `S` — Lock down the agent server's CORS + auth.** ~~`agent-server.ts:231` sets `Access-Control-Allow-Origin: *` and `isAuthorized()` (line 828) returns `true` whenever no `HELMSTACK_AUTH_TOKEN` is set (the default).~~ **Done:** `main.ts` now generates a token on first launch (`readOrCreateAgentToken`, persisted 0600 to `helmstack-agent-token`, printed to the console) so auth is on by default; `agent-server.ts` adds a loopback-only `Host` guard (DNS-rebinding), a cross-origin `Origin` guard, and echoes CORS only for loopback origins (no more `*`). - [ ] **`P0` `S` — Agent isolation is advisory only.** Tab ownership keys off the client-supplied `X-Agent-ID` header (`agentIdOf`, line 211). Any client can impersonate any `agentId` and reach another agent's tabs/approvals/handoffs. Either bind agent identity to the auth token or document loudly that multi-agent isolation is not a security boundary.
- [x] **`P1` `S` — REST inputs are unvalidated.** **Done:** added `request-validation.ts` — zero-dependency, pure validators (`parseNetworkMockRules`, `parseResourceBudget`, `parseAccountInput`, `parseViewportBody`) returning a typed value or a pointed error. Wired into the four endpoints that previously cast complex nested bodies (`POST /accounts`, `/mock`, `/budget`, `/viewport`), so malformed input now yields a real `400` (e.g. `rules[0].responseStatus must be a number`) instead of a later crash. (Used hand-rolled validators rather than pulling `zod` into the Electron bundle — it's only a hoisted transitive dep; the other endpoints already inline-check their scalar bodies.) Covered by `apps/desktop/test/request-validation.test.ts` (9 cases).
- [x] **`P1` `S` — Anti-detection jitter degrades dev/test runs.** **Done:** added `runtime-config.ts` `isStealthEnabled()` (env `HELMSTACK_STEALTH`, default off). `humanJitter()` is now a no-op unless stealth is on, so default click/type/select/submit have zero added latency; the anti-detection fingerprint injection in `tab-manager.ts` is likewise gated behind the same flag. Covered by `apps/desktop/test/runtime-config.test.ts`.
- [x] **`P1` `M` — In-memory-only baselines.** **Done:** added `baseline-store.ts` with `ScreenshotBaselineStore` (PNG bytes → `helmstack-screenshots/<sha1>.png` + `index.json`) and `PerceptionBaselineStore` (PageGraph → `helmstack-perception/<sha1>.json` + `index.json`), both under `userData`. `TabManager` rehydrates both caches on construction and writes through on `captureNamedScreenshot`/`saveNamedPerception` + removes on delete, so baselines survive restarts. - [x] **`P2` `S` — Inconsistent `debugger.isAttached()` guards.** **Done:** added a private `ensureDebugger(webContents)` helper that attaches if needed and returns the debugger; replaced all 20 duplicated inline attach blocks with it and added it to the four methods that previously called `Runtime.evaluate` with no guard at all (`inspectElementStyles`, `getStorage`, `setStorage`, `clearStorage`). The only remaining `isAttached()` calls are two intentional read-only conditionals (viewport-suite restore + a teardown path).
- [x] **`P2` `S` — `approveCommand` is O(tabs) with a perception capture per tab.** **Done:** `approveCommand` now resolves the owning tab directly from the pending approval (`listPendingApprovals().find(...)` → `pending.tabId`) and captures perception only for that one tab, instead of probing every tab. Returns a clear `failed` result if the owning tab has since closed.
- [x] **`P2` `S` — SSE has no keep-alive.** **Done:** each `/api/stream` client now gets a 25s `: ping` comment-frame heartbeat (`agent-server.ts`), cleared on `close` and on write failure (which also evicts the dead client); `interval.unref()` so it never holds the process open.

## 2. Perception layer — make it developer-relevant
